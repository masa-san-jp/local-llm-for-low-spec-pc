## v0.1 — 2026-04-10

### 背景・目的

低スペックPC向けローカルLLMアプリの利便性向上を目的として、以下の改善を計画します。  
方針：
- ユーザーが簡単に使用できる選択肢の提供（Docker・ターミナルセットアップの両対応）
- 初心者・プログラミング未経験者に配慮した構成（自動化・一括セットアップ・ヘルプの提供）

---

### 改善項目

#### 1. Dockerセットアップの提供 ⬜ 未着手

Dockerを利用することで、ローカル環境に依存せず利用者が簡便にプロジェクトを実行できる環境を構築する。

**要件**
- プロジェクト動作用の `Dockerfile` を作成し、Dockerイメージを生成可能にする
- Docker Hubに公開済みイメージを提供する（GitHub Actionsで自動ビルド対応）
- `README.md` にDockerセットアップ手順を追記

**タスク**
- [ ] `Dockerfile` をプロジェクトルートに作成
- [ ] GitHub ActionsのワークフローでDockerイメージを自動ビルド＆Push
- [ ] Dockerイメージに必要なPythonライブラリとモデルの依存関係を含める

---

#### 2. 簡易なターミナルセットアップの提供 ⬜ 未着手

Dockerを利用しない環境・ユーザー向けに、直感的かつ自動化されたセットアップスクリプトを提供する。

**要件**
- 一括でPythonの依存関係をインストールするシェルスクリプト（`setup.sh`）およびWindows用バッチファイル（`setup.bat`）を作成
- 必要なPythonライブラリとツールのインストールを自動化
- モデルデータの自動ダウンロード機能
- インストール完了後にプロジェクト起動の簡単な案内を表示

**タスク**
- [ ] `setup.sh` と `setup.bat` をリポジトリに追加
- [ ] `requirements.txt` を整備し、確実に動作する依存関係管理を行う

---

#### 3. トラブルシューティングの整備 ⬜ 未着手

セットアップや運用時に初心者がエラーでつまずかないよう、リソースとヘルプを提供する。

**要件**
- よくあるトラブルケース（エラーメッセージや発生原因）への対処例をREADME内に記載
- FAQセクションを追加し、簡潔な解決手順を説明
- GitHub Discussionsを有効化し、コミュニティサポートの場を提供

**タスク**
- [ ] FAQセクションを作成（モデルダウンロード失敗時・Docker未インストール時など）
- [ ] README内にエラー時の案内セクションを追加
- [ ] GitHub Discussionsをリポジトリ設定から有効化

---

#### 4. READMEのリファクタリング ⬜ 未着手

全体のユーザーガイドを見直し、分かりやすく適切な内容に再構成する。

**要件**
- 「Dockerによるセットアップ」「ターミナルセットアップ」の2通りの手順を明示
- セットアップ難易度や推奨利用シーンを説明
- すべてのコマンド例を明確化し、コードブロックに整理

**タスク**
- [ ] 現行のREADMEにセットアップセクションを追記・修正
  - 1. プロジェクトの目的と概要
  - 2. セットアップオプション（Dockerセットアップ推奨 / ターミナルセットアップ）
  - 3. 実行方法
  - 4. トラブルシューティング

---

#### 5. GitHub ActionsによるCI/CDの実装 ⬜ 未着手

リポジトリ更新時にセットアップが正しく動作することを保証するため、継続的インテグレーション（CI）を導入する。

**要件**
- 毎回のリポジトリ更新でPython環境での依存関係インストール確認・テストスクリプト実行を行う
- マージ後にDockerイメージを自動ビルド＆Docker HubへPush

**タスク**
- [ ] `.github/workflows/ci.yml` を作成
  - Python環境のテスト（`requirements.txt` を基に仮想環境でのinstall/checkを実行）
  - Dockerイメージの生成とプッシュ

### 期待される成果

1. **簡易セットアップの提供**: ユーザー自身がスムーズにセットアップと実行を完了できる。プログラミングスキルに関わらず、プロジェクトの利用を可能にする。
2. **ユーザー数の拡大**: 初心者でも問題なく利用できる環境を整備することで対象ユーザーが拡大。エラー発生時の対処法の明示化により、サポートコストを削減。
3. **保守性の向上**: CI/CDパイプラインを導入し、コードや環境更新時のトラブルを未然に防ぐ。Dockerコンテナの提供により、動作環境のばらつきを排除。
