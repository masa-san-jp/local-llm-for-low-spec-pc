## v0.3 — 2026-04-10

### 背景・目的

ユーザーがファイルを添付したとき、PDFのテキストが正しく抽出されない・CSVの日本語が文字化けする・どのファイル形式が使えるかわからない、という3つの問題が報告された。  
ファイル添付機能を実用レベルに改善し、ユーザーの混乱を解消することを目的とする。

---

### 改善項目

#### 1. PDF テキスト抽出の修正 ✅ 完了

**問題**  
PDFファイルを添付するとモデルに raw なバイナリオブジェクトコードが渡され、LLM がテキストを読めなかった。

**原因**  
`extractPdfText` が `FileReader.readAsText` + 正規表現によるノイズ除去という簡易実装であり、実際の PDF 構造解析を行っていなかった。

**対処**  
- `pdfjs-dist` ライブラリを導入し、`getDocument` / `getTextContent` API による正確なテキスト抽出を実装
- 抽出結果が 10 文字未満の場合（スキャン PDF 等）はエラーメッセージを表示
- コンテキスト超過防止のため抽出テキストを 8,000 文字に制限（従来どおり）

**タスク**
- [x] `pdfjs-dist` を `npm install` で追加
- [x] `fileProcessor.ts` の `extractPdfText` を pdfjs-dist ベースに書き換え
- [x] PDF.js の Web Worker を Vite の `new URL(...)` 構文で設定

---

#### 2. CSV・テキストファイルの文字コード自動判別 ✅ 完了

**問題**  
Shift-JIS エンコードの CSV ファイルを添付すると日本語が文字化けし、LLM に正しい内容が渡らなかった。

**原因**  
`fileToText` が `FileReader.readAsText(file, 'utf-8')` で固定的に UTF-8 読み込みをしていた。

**対処**  
- `File.arrayBuffer()` でバイト列を取得し、`TextDecoder('utf-8')` で変換後に置換文字（U+FFFD）が含まれていれば `TextDecoder('shift-jis')` で再変換する自動判別ロジックを実装
- 置換文字数が少ない方のデコード結果を採用することで誤検知を抑制

**タスク**
- [x] `fileProcessor.ts` の `fileToText` を `ArrayBuffer` + `TextDecoder` ベースに書き換え
- [x] UTF-8 → Shift-JIS のフォールバック処理を追加

---

#### 3. 添付可能なファイル形式の明示 ✅ 完了

**問題**  
どのファイル形式を添付できるかユーザーが把握できず、混乱が生じていた。

**対処**  
- 入力エリア上部に「添付可能: 画像 (JPG/PNG/GIF/WebP/HEIC)・テキスト (TXT/MD/CSV)・PDF」の説明テキストを追加
- 添付ボタンの `title` 属性を「画像・テキスト・CSV・PDF」に更新

**タスク**
- [x] `ChatInput.tsx` にサポートファイル形式の案内テキストを追加
- [x] 添付ボタンのツールチップを更新
